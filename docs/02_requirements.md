Этап 2. Требования (MVP) + Нефункциональные требования
Файл: docs/02_requirements.md

MUST (Обязательные требования):

Поиск города: Пользователь должен иметь возможность ввести название города в текстовое поле.

Критерий приёмки: При вводе "Москв" в выпадающем списке появляется "Москва".

Текущая погода: Приложение должно отображать текущую температуру, влажность, скорость ветра и описание погоды для выбранного города.

Критерий приёмки: После выбора города, на главном экране обновляются все указанные метрики.

Прогноз: Приложение должно отображать прогноз погоды на 3 дня (дата, мин/макс температура, иконка/описание).

Критерий приёмки: Под блоком текущей погоды отображаются 3 карточки с данными на следующие дни.

Переключение единиц: Пользователь может переключить отображение температуры в градусах Цельсия или Фаренгейта в настройках.

Критерий приёмки: После смены настройки с C на F, температура на всех экранах пересчитывается и отображается в F.

Избранное (добавление): Пользователь может добавить текущий отображаемый город в список избранного.

Критерий приёмки: Рядом с названием города есть кнопка "Добавить в избранное". После нажатия город появляется в списке "Избранное".

Избранное (просмотр): Пользователь может кликнуть на город в списке избранного, чтобы загрузить его погоду.

Критерий приёмки: При клике на "Лондон" в списке избранного, главный экран переключается на отображение погоды в Лондоне.

Сохранение настроек: Выбранные единицы измерения и список избранных городов должны сохраняться между запусками приложения.

Критерий приёмки: После перезапуска приложения в настройках стоит C/F, выбранные пользователем ранее, а список избранного не пуст.

NICE (Желательные требования):

Автодополнение (подсказки) при вводе города.

Отображение времени последнего обновления данных.

Кнопка "Обновить" для принудительного запроса данных с сервера.

Индикация "облачности" или иконки погоды (солнечно/пасмурно/дождь).
