Этап 7. Конфигурация, хранение, ограничения
Файл: docs/07_config_storage_constraints.md

Конфиг-параметры (по умолчанию):

units: "C"

cache_ttl_minutes: 10

forecast_days_default: 3

network_timeout_seconds: 5

Где храним локальные файлы:

Windows: %APPDATA%/WeatherApp/

Linux: ~/.config/WeatherApp/

Файлы: favorites.json, settings.json. Кэш в оперативной памяти (in-memory).

Политика кеша:

Кэшируем: CurrentWeather и List[DailyForecast].

Ключ кеша: название города + тип данных (current/forecast).

TTL: 10 минут (значение из конфига).

Ограничения/лимиты:

API: Бесплатный тариф погодного API (например, 60 запросов в минуту).

Платформа: Не более 5 запросов в секунду к одному хосту (чтобы избежать троттлинга на уровне ОС/библиотек).

Политика ошибок:

Пользователю показываем текст из этапа 3 (В.4).

Логируем: стектрейс ошибки (если это баг), код ответа API (если 404, 500), тип ошибки (NetworkError) в консоль или файл лога.
