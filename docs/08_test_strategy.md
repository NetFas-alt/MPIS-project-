Unit-тесты:

Конвертация температуры C <-> F.

Валидация входных данных (пустой город, специальные символы).

Форматирование вывода (даты, числа).

Интеграционные тесты:

Маппинг DTO (сырой JSON от API) в Domain-модели.

Работа с мок-сервером: эмуляция ошибок сети (таймаут), 404 (город не найден), 429 (лимиты).

Проверка чтения/записи JSON-файлов (фикстуры).

E2E/Acceptance:

Smoke-тест: запуск приложения, проверка, что главное окно открывается.

Сценарий: поиск города "London" -> проверка, что данные на экране не пустые.

Сценарий: добавление города в избранное -> перезапуск приложения -> проверка, что город остался в списке.

Матрица MUST -> тесты:

MUST "Поиск города" -> E2E тест (ввод, проверка выпадающего списка).

MUST "Сохранение настроек" -> Интеграционный тест (запись в файл) + E2E тест (перезапуск приложения).
